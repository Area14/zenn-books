## 単体で使う vs 連携させる

ここまで、5人のAIエージェントを個別に活用してきました。秘書に議事録を作らせ、リサーチャーに市場調査を頼み、経理に経費を集計させ、営業に提案書を書かせ、マーケターにSNS投稿を量産させる。それぞれが単独で十分に価値を発揮してくれました。

しかし、本当のチームとは、1人ひとりが優秀なだけでは成り立ちません。**メンバー同士が連携して、1人ではできなかった仕事をチームとして達成する**。そこに、チームの真の力があります。

5人のAIエージェントを連携させれば、ひとつの大きな仕事を一気通貫で処理できる。その方法を、実際に体験します。


## 連携の基本：Slackスレッドでのリレー方式

現時点でのAIエージェント連携は、「あなたが中継役になる」方式が基本です。

仕組みはシンプルです。

1. `#ai-research` で市場調査レポートを作成
2. その結果をコピーして `#ai-marketing` に貼り付け、マーケティング施策案を作成
3. さらに `#ai-sales` に貼り付けて、営業メールに展開

あなたが「受け渡し役」を担うことで、各チャンネルのエージェントが協力して仕事を進めます。

「え、手動でコピペするの？」と思ったかもしれません。確かに、理想的には「リサーチが終わったら自動的にマーケティングに渡す」という全自動パイプラインが欲しいところです。Claude Managed Agentsのマルチエージェント機能は現在リサーチプレビュー段階にあり、今後さらに洗練されていく見込みです。

しかし、「あなたが中継する」方式には、実は大きなメリットがあります。

**各ステップで中身を確認できる。**

全自動は便利ですが、途中でAIが間違った方向に進んでいても気づけません。あなたが中継役になることで、各段階の出力をチェックし、必要に応じて軌道修正できます。これは、特にAIエージェントを使い始めたばかりの段階では、非常に重要です。


## ハンズオン⑱ 一気通貫のリレーワーク

それでは、5人のAIエージェントをリレーさせて、「新サービスの立ち上げ」を一気に進めてみます。

### ステップ1：リサーチ（#ai-research）

```
@Claude 中小企業向けの「AIを活用したWebサイト改善コンサルティング」
というサービスを企画しています。

以下を調査してください：
1. 中小企業のWebサイト活用の現状と課題
2. AIを使ったWeb改善サービスの競合状況
3. 想定ターゲットのニーズ（何に困っているか）
4. 適切な価格帯の参考情報

エグゼクティブサマリー + 詳細の形式でレポートをお願いします。
```

**→ レポートが返ってきたら、エグゼクティブサマリー部分をコピーします。**

### ステップ2：マーケティング企画（#ai-marketing）

```
@Claude 以下のリサーチ結果をもとに、新サービス
「AI×Web改善コンサルティング」のマーケティング施策案を作ってください。

【リサーチ結果（要約）】
（ステップ1のエグゼクティブサマリーを貼り付け）

作成してほしいもの：
1. サービスのキャッチコピー案（5パターン）
2. ターゲットペルソナ（想定顧客の具体像）
3. 集客チャネル戦略（どこでどう告知するか）
4. LP（ランディングページ）の構成案
5. ローンチ後1ヶ月のSNS投稿カレンダー
```

**→ キャッチコピーとペルソナ情報をコピーします。**

### ステップ3：営業ツール作成（#ai-sales）

```
@Claude 以下の新サービスについて、営業用の提案書ドラフトを作成してください。

【サービス概要】
サービス名：AI×Web改善コンサルティング
キャッチコピー：（ステップ2で選んだコピーを貼り付け）
ターゲット：（ステップ2のペルソナを貼り付け）

提案書に含めること：
1. サービス概要
2. 解決できる課題
3. サービスの流れ（4ステップ）
4. 料金プラン（3段階）
5. 想定される成果
6. 次のアクション

さらに、既存顧客5社への案内メール（個別カスタマイズ版）も作成してください。
```

**→ 料金プランをコピーします。**

### ステップ4：経理シミュレーション（#ai-accounting）

```
@Claude 新サービスの収支シミュレーションを作成してください。

【条件】
- 料金プラン：（ステップ3の料金プランを貼り付け）
- 初月の想定顧客数：2社
- 3ヶ月後の想定顧客数：5社
- 6ヶ月後の想定顧客数：10社
- 固定コスト：AIツール月額$100、その他月2万円
- 変動コスト：1社あたり作業時間10時間、時給換算5,000円

出力：
- 6ヶ月間の月次収支予測表
- 損益分岐点
- 年間売上予測
```

**→ 収支表をコピーします。**

### ステップ5：秘書が全体をまとめる（#ai-secretary）

```
@Claude 新サービス「AI×Web改善コンサルティング」の企画サマリーを
作成してください。経営判断用の1ページ資料です。

以下の情報をまとめてください：
- 市場環境：（ステップ1の要約）
- サービス概要とポジショニング
- 収支見通し：（ステップ4の要約）
- ローンチまでのToDoリスト
- 判断が必要な事項

A4 1ページに収まる分量で、結論ファーストでお願いします。
```

### 全体の流れ

```
リサーチ → マーケティング → 営業 → 経理 → 秘書（統合）
   #ai-research  #ai-marketing  #ai-sales  #ai-accounting  #ai-secretary
```

5つのチャンネルを横断して、約30分〜1時間で「新サービスの企画書一式」が出来上がります。これを人間だけでやろうとすれば、少なくとも1〜2週間はかかる仕事です。

:::message
**5エージェント連携フロー**

以下は、5人のAIエージェントが連携する全体像です。

```
┌─────────────────────────────────────────────────────┐
│  あなた（ひとり社長）＝ 中継役・判断者              │
│                                                     │
│  ① テーマを投げる                                   │
│       ↓                                             │
│  #ai-research  ──→ 市場調査レポート                 │
│       ↓ （要点を抜粋してコピー）                    │
│  #ai-marketing ──→ マーケ施策・LP構成案             │
│       ↓ （キャッチコピー＋ペルソナを抜粋）          │
│  #ai-sales     ──→ 提案書・営業メール               │
│       ↓ （料金プランを抜粋）                        │
│  #ai-accounting──→ 収支シミュレーション             │
│       ↓ （各成果物の要約を集約）                    │
│  #ai-secretary ──→ 経営判断用サマリー（A4 1枚）     │
│                                                     │
│  各ステップであなたが内容を確認 → GO / やり直し判断 │
└─────────────────────────────────────────────────────┘
```

ポイントは「全文を渡さない」ことです。各ステップの出力から要点だけを抜き出して次に渡すことで、エージェントが情報に埋もれず、質の高い出力を返してくれます。
:::

もちろん、AIが生成したものをそのまま最終成果物にするわけではありません。しかし、「叩き台」が揃っている状態と、白紙から始める状態では、仕上げの労力がまるで違います。


## 連携をスムーズにするコツ

### コツ1：情報の「圧縮」を意識する

あるチャンネルの出力を、次のチャンネルに渡す際、**全文をコピペする必要はありません**。エグゼクティブサマリーや要点だけを抜き出して渡す方が、次のエージェントの出力品質が上がります。

情報が多すぎると、AIは「何が重要か」を見失いがちです。あなたが「ここが重要」と判断した部分だけを渡すことで、次のステップの精度が上がります。

### コツ2：各ステップで「GO / NO-GO」を判断する

リレーの各段階で、あなたが中身を確認します。「リサーチ結果がイマイチだな」と思ったら、その段階でやり直す。マーケティング施策の方向性が違うと感じたら、修正を指示する。

全自動ではないからこそ、あなたの経営判断を各段階に反映できるのです。

### コツ3：よく使うリレーパターンを定型化する

「リサーチ → マーケティング → 営業」のようなリレーパターンが定着してきたら、各ステップのプロンプトをテンプレートとしてSlackのブックマークに保存しておくと便利です。次回からは、テンプレートの一部を書き換えるだけで同じ流れを再現できます。
