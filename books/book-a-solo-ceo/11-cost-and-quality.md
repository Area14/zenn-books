## AIチームの「人件費」を把握する

会社を経営する上で、コスト管理は避けて通れません。AIエージェントは人間のスタッフよりはるかに安いとはいえ、「使い放題」ではありません。料金体系を理解し、コストを最適化する方法を押さえておきます。


## 料金体系の仕組み

本書で使う「Claude Code in Slack」のコストは、シンプルな月額定額制です。

### Claude Pro / Max プランの月額料金

第3章で契約したプランの月額料金が、すべてのコストです。

| プラン | 月額 | 利用量の目安 |
|:---|:---|:---|
| Pro | $20（約3,000円） | 基本的な利用には十分 |
| Max 5x | $100（約15,000円） | 日常的にヘビーに使う場合 |
| Max 20x | $200（約30,000円） | 大量のエージェント処理が必要な場合 |

**Slack経由でClaude Code in Slackを使う場合は、プランの月額料金に含まれています。** 追加料金は発生しません。各セッション（AIエージェントが作業する1回分）は、プランに含まれる利用枠から消費されます。

:::message alert
**利用上限について**
各プランには利用上限があります。Proプランで集中的に使うと、上限に達して「Rate limit reached」（利用上限到達）と表示されることがあります。この場合、しばらく待つか（通常数時間でリセット）、上位プランにアップグレードすることで対応できます。上限に達しても、追加課金が自動的に発生することはありません。
:::

### 参考：Managed Agents API の料金（開発者向け）

本書では使いませんが、参考情報として記載します。開発者がManaged Agents APIを直接プログラムから利用する場合は、月額プランとは別に、トークン従量課金（テキスト量に応じた料金）とセッション時間料金（$0.08 / session-hour）がかかります。本書のSlack経由の利用とは別の課金体系ですので、混同しないよう注意してください。

つまり、ひとり社長がSlackから日常的にAIエージェントを使う分には、**月額$20〜$100の固定費で収まる**ということです。


## 月額コストのシミュレーション

具体的な利用パターンでコストを計算します。

### パターン1：ライトユース（月$20 = 約3,000円）

| 項目 | 頻度 |
|:---|:---|
| AI秘書への依頼 | 週3〜5回（議事録、メール下書き） |
| リサーチ依頼 | 週1回 |
| 経費集計 | 月1回 |
| 営業資料作成 | 月2〜3回 |
| SNS投稿案作成 | 月1回（まとめて） |

Proプラン（$20/月）で十分にカバーできる利用量です。始めたばかりの方はこのパターンで十分。月約3,000円で5人のAIチームが動きます。

**費用対効果**：月10時間の作業時間を節約できたとすると、時給3,000円換算で月30,000円相当。投資対効果は10倍です。

### パターン2：ミディアムユース（月$100 = 約15,000円）

| 項目 | 頻度 |
|:---|:---|
| AI秘書への依頼 | 毎日1〜2回 |
| リサーチ依頼 | 週2〜3回 |
| 経費・売上分析 | 週1回 |
| 営業資料・メール | 週3〜5回 |
| コンテンツ作成 | 週2〜3回（ブログ、SNS、LP） |

Max 5xプラン（$100/月）が適しています。AIエージェントを業務の中核に据えて、本格的に活用するパターンです。

**費用対効果**：月40時間の作業時間を節約。時給3,000円換算で月120,000円相当。投資対効果は8倍です。

### パターン3：ヘビーユース（月$200 = 約30,000円）

| 項目 | 頻度 |
|:---|:---|
| 全エージェントを毎日フル活用 | — |
| 大規模なリサーチプロジェクト | 週1回以上 |
| LP制作やWebアプリ生成 | 月数回 |
| 大量のコンテンツ生成 | 毎日 |

Max 20xプラン（$200/月）。複数のクライアント案件を同時に回すような、パワーユーザー向けです。

### コスト比較：AI vs 人間 vs 外注

| | AI（Pro） | AI（Max 5x） | パートタイム社員 | 外注 |
|:---|:---|:---|:---|:---|
| 月額 | 約3,000円 | 約15,000円 | 約15〜20万円 | 案件ごと |
| 対応時間 | 24時間 | 24時間 | 月80時間程度 | 依頼時のみ |
| 即応性 | 数秒〜数分 | 数秒〜数分 | 勤務時間内 | 数日 |
| 対応範囲 | 5部署分 | 5部署分 | 1〜2領域 | 1領域 |

## 品質を上げる：ロールプロンプトの改善サイクル

AIエージェントの出力品質は、ロールプロンプトの質に大きく依存します。最初に設定したロールプロンプトが完璧であることは稀で、使いながら少しずつ改善していくのが正しいアプローチです。

### PDCAサイクルで改善する

**Plan（計画）**：こういう出力がほしい、という期待を明確にする
**Do（実行）**：エージェントにタスクを依頼する
**Check（確認）**：出力が期待通りだったか評価する
**Act（改善）**：ロールプロンプトに改善点を反映する

### よくある改善パターン

| 問題 | ロールプロンプトに追記する内容 |
|:---|:---|
| 回答が長すぎる | 「回答は300文字以内に収めてください」 |
| 結論が最後に来る | 「結論ファーストで報告してください」 |
| 表形式にしてくれない | 「数値を含む情報は必ず表形式で出力」 |
| 敬語が堅すぎる | 「丁寧語だが、堅くなりすぎないトーンで」 |
| 余計な前置きが長い | 「挨拶や前置きは省略し、本題から始めて」 |
| 重要度が分からない | 「重要度を高・中・低で明示してください」 |

### 改善のコツ

1つずつ変更する。一度に5つも6つも変えると、どの変更が効果的だったか分からなくなります。

1週間に1回、ロールプロンプトを見直す時間を設けてください。金曜の夕方15分で十分です。「今週、AIの出力に不満だった点」をメモしておき、それをロールプロンプトの改善に反映します。


## 月次のエージェント運用レビュー チェックリスト

毎月末に、以下の項目をチェックしてください。

**コスト面**
- [ ] プランの利用上限に達していないか
- [ ] プランのアップグレード/ダウングレードは必要か
- [ ] コストに見合った価値を得られているか

**品質面**
- [ ] 各エージェントのロールプロンプトは最新か
- [ ] 出力品質に不満はないか
- [ ] 新しく任せたいタスクはないか

**運用面**
- [ ] 使っていないチャンネルはないか
- [ ] セキュリティ上の懸念はないか
- [ ] 朝のタスク整理ルーティンは継続できているか

**成長面**
- [ ] AIに任せる業務の範囲は広がっているか
- [ ] プロンプトの書き方は上達しているか
- [ ] AIとの協業で新しい発見はあったか


## エージェントの「評価」方法

ロールプロンプトを改善するには、まず現在の品質を正しく評価する必要があります。以下の3つの基準で、各エージェントの出力を定期的にチェックしてください。

| 評価基準 | チェック内容 | 合格ライン |
|:---|:---|:---|
| 正確さ | 事実関係に誤りがないか、数値は正しいか | 重大な事実誤認がゼロ |
| 実用性 | そのまま使える品質か、手直しがどの程度必要か | 手直し10分以内 |
| 一貫性 | 毎回安定した品質か、ムラがないか | 10回中8回以上が合格 |

評価が低い項目があれば、原因はほぼ確実にロールプロンプトにあります。前述のPDCAサイクルで改善を回してください。
