## なぜSlackをインターフェースにするのか

AIエージェントに指示を出す方法は、実はいくつかあります。

Anthropicが提供する「Claude Console」というWebサイト上の管理画面を使う方法。プログラマー向けのAPI（プログラムからの呼び出し口）を使う方法。そして、Slackを使う方法です。

本書では、迷わずSlackを選びます。その理由は3つあります。

**理由1：いつものツールで完結する**

ビジネスでSlackを使っている方は多いはずです。使い慣れたツールの中でAIに指示を出せるということは、新しいツールの使い方を覚える必要がないということです。ブラウザで管理画面を開いて、あちこちのボタンをクリックして——という手間が一切ありません。Slackを開いて、メッセージを打つ。それだけです。

**理由2：スマホからでも指示が出せる**

Slackはスマートフォンアプリがあります。つまり、通勤電車の中からでも、カフェからでも、お風呂上がりにソファに座りながらでも、AIエージェントに仕事を頼めるということです。「あ、あの件、エージェントにやらせておこう」と思いついた瞬間に、ポケットのスマホから指示を飛ばせます。

**理由3：会話のログが自然に残る**

Slackのスレッド（返信の連なり）がそのまま仕事の記録になります。「いつ、何を頼んで、どんな結果が返ってきたか」がすべてSlack上に残る。あとで見返すのも、検索するのも簡単です。メールのやり取りを探すよりも、はるかに効率的です。


## 5つのチャンネル ＝ 5つの部署

会社にはいくつかの「部署」があります。それぞれの部署には、それぞれの専門性と担当業務があります。

本書では、Slackのチャンネル（話題ごとの部屋）を「部署」に見立てます。5つのチャンネルを作り、それぞれにAIエージェントを配置する。これが、あなたの「AIオフィス」です。

### `#ai-secretary`（秘書室）

経営者の右腕となるエージェントが常駐するチャンネルです。議事録作成、タスク整理、メール下書き、資料のアウトライン作成など、「ちょっとこれお願い」と言いたくなる雑多な仕事を引き受けてくれます。「何でも屋」として最も頻繁に使うチャンネルです。

### `#ai-research`（リサーチ部）

情報収集と分析を専門とするエージェントのチャンネルです。市場調査、競合分析、業界トレンドの整理などを依頼すると、Web上の情報を収集・整理してレポートにまとめてくれます。調べ始めると際限なく溶けるリサーチ時間を大幅に圧縮できます。

### `#ai-accounting`（経理部）

お金まわりの管理を担当するチャンネルです。CSVファイル（表形式のデータ）をSlackにアップロードして「先月の売上を集計して」と頼めば、AIが自動でプログラムを書いて実行し、集計結果をまとめてくれます。あなたがプログラムを書く必要は一切ありません。

### `#ai-sales`（営業部）

営業活動を支援するチャンネルです。提案書の自動生成、顧客リストの整理、フォローメールの一括作成など、営業の前後にある書類仕事を引き受けてくれます。あなたはお客様との対話に集中できます。

### `#ai-marketing`（マーケティング部）

集客と情報発信を担当するチャンネルです。SNS投稿案の一括生成、ブログ記事の下書き、広告コピー、ランディングページ（LP）の制作まで対応します。マーケティングはひとり社長が「手が回らない」仕事の筆頭ですが、AIに任せれば定期的な発信が続けられます。


## Claude Code in Slack の仕組み

SlackからAIエージェントが動く仕組みを、もう少し詳しく見ておきます。難しい技術の話はしませんが、全体像を知っておくと使いこなしやすくなります。

Slackの中で`@Claude`とメンションしてメッセージを送ると、裏側で次のようなことが起きています。

```
あなたがSlackで指示を出す
　　↓
① Claude がメッセージを受け取る
　　↓
② 「これはコーディング（作業系）の依頼だな」と判断する
　　↓
③ Claude Code のセッション（作業場）が自動で作られる
　　↓
④ エージェントが作業を開始する
　　↓
⑤ 途中経過がSlackのスレッドに投稿される
　　↓
⑥ 作業が完了すると、結果をSlackに報告してくれる
```

ポイントは、**② の自動判断**です。Claudeは、あなたのメッセージを読んで「これは単なる雑談か、それともファイル操作やコード実行が必要な作業依頼か」を自動で判別します。作業が必要だと判断すれば、自動的にClaude Codeというパワフルな実行環境にバトンタッチされます。

これは、受付スタッフが電話を受けて「この件は経理部に回しますね」と内線を転送してくれるようなものです。あなたは、誰に転送されるかを気にする必要はありません。ただ、やってほしいことを伝えるだけです。

作業が完了すると、Slackスレッドに「完了しました」というメッセージと一緒に、いくつかのボタンが表示されます。

- **View Session（セッションを見る）** — 作業の全過程をブラウザで確認できます
- **Create PR** — 作成したコードやファイルの変更をまとめることができます

特に「View Session」は便利です。エージェントが「何をどう考えて、どんな手順で作業したか」の全記録が見られるので、結果に疑問があった場合も、作業プロセスを遡って確認できます。


## Claudeの3つの入口と本書の選択

Anthropicが提供するAIエージェント関連のサービスには、Slack（本書で全面的に使用）、Claude Console（claude.aiの管理画面、設定確認時のみ使用）、Managed Agents API（エンジニア向け、本書では扱わない）の3つの入口があります。

本書で使うClaude Code in Slackは、ProやMaxプランの月額料金に含まれており、追加料金もプログラミングも不要です。さらに、Slackのスレッド内で@Claudeをメンションすると、それまでの会話を踏まえて作業を開始してくれます。
