## はじめての @Claude

セットアップが完了しました。あなたのSlackには、5つの部署チャンネルが並び、それぞれにClaudeが招待されています。

さっそく最初の会話を始めます。

まずは `#ai-secretary`（秘書室）チャンネルを開いてください。ここが、あなたとAIエージェントの最初の「面談」の場所です。


## ハンズオン⑤ 「自己紹介文を作って」と頼んでみる

最初のタスクとして、シンプルだけど実用的なお願いをします。

**Slackの `#ai-secretary` チャンネルで、以下のメッセージを送信してください：**

```
@Claude 私はWebデザインのフリーランスをしています。
新規のお客様向けに、自己紹介文を作ってください。

条件：
- 300文字程度
- 経験年数は5年
- 得意分野はコーポレートサイトとECサイト
- 丁寧だけど親しみやすいトーン
```

Enterキーを押して送信すると、数秒後にClaudeから反応があります。

### 何が起きているか

メッセージを送信した瞬間から、裏側では次のことが起きています。

1. **Claudeがメッセージを受け取る** — あなたの@メンションを検知します
2. **内容を分析する** — 「文章を作成してほしい」という依頼だと理解します
3. **ルーティング判断** — この内容なら、チャット応答で十分か、Claude Codeのセッションを立ち上げるかを判断します
4. **応答を生成する** — 指定した条件に沿って自己紹介文を作成します
5. **Slackに返信する** — スレッド内に結果を投稿します

今回のような「テキストを生成するだけ」のタスクであれば、チャット応答で即座に返ってくることが多いです。一方、「ファイルを作成して」「データを集計して」のような作業系のタスクでは、Claude Codeセッションが起動し、進捗が随時報告されます。

### 返ってきた結果を確認する

Claudeが返してくれた自己紹介文を読んでみてください。おそらく、そのまま使えるレベルの文章が返ってきたはずです。

でも、「もう少しカジュアルにしてほしい」「実績の数字を入れたい」と思ったかもしれません。そんなときは、**スレッドで追加の指示を出します**。


## ハンズオン⑥ スレッドで追加指示を出す

Claudeの返信にカーソルを合わせ、「スレッドで返信」をクリックしてください。

**スレッド内で、以下のメッセージを送信します：**

```
@Claude ありがとう。いい感じですが、以下の修正をお願いします。

- もう少しカジュアルなトーンにして（「です・ます」は残しつつ）
- 制作実績として「累計50サイト以上」を追加
- 最後に「お気軽にご相談ください」で締めて
```

スレッドの便利なところは、**Claudeが前のメッセージの文脈を覚えている**ことです。最初に依頼した自己紹介文の内容を覚えた上で、修正を加えてくれます。一から説明し直す必要はありません。

これは、人間の秘書に「さっきの文章、ここを直して」と頼むのと同じ感覚です。

修正版が返ってきたら、それをそのままコピーして使えます。もちろん、さらに修正を重ねることも可能です。何度でもスレッド内でやり取りを続けられます。


## セッションの流れを理解する

ここで、Claude Code in Slackの「セッション」という概念を整理しておきます。

### チャット応答とClaude Codeセッションの違い

| | チャット応答 | Claude Codeセッション |
|:---|:---|:---|
| 起動条件 | 質問、相談、テキスト生成 | ファイル操作、コード実行、Web調査 |
| 応答速度 | 数秒〜十数秒 | 数十秒〜数分 |
| Slack上の表示 | 直接返信される | 進捗メッセージが随時投稿される |
| 詳細確認 | スレッドを見るだけ | 「View Session」ボタンで全過程を確認 |
| できること | テキストの生成・編集 | ファイル作成、データ処理、Web検索、コード実行 |

簡単な質問やテキスト生成は「チャット応答」で済みます。一方、本書の後半で扱うような「CSVを集計してグラフを作れ」「Webサイトを調べてレポートにまとめて」といった複雑なタスクでは、Claude Codeセッションが起動します。

### セッションが起動したときの流れ

Claude Codeセッションが起動すると、Slackのスレッドに進捗メッセージが投稿されます。

```
Claude Code session started...
（セッション開始）

Reading files and analyzing data...
（ファイルを読み込み、データを分析中）

Task completed. View the full session for details.
（タスク完了。詳細はセッションを確認してください）
```

このようなメッセージが随時投稿されるので、エージェントが今何をしているかがリアルタイムで分かります。

タスク完了後に表示される「View Session」ボタンをクリックすると、ブラウザで `claude.ai/code` が開き、作業の全記録を確認できます。エージェントがどんな手順で作業したか、途中でどんな判断をしたか、すべてが記録されています。


## 「良い指示」と「悪い指示」

AIエージェントは、あなたの指示の質によって、返してくる結果の質が大きく変わります。指示の出し方のコツを押さえておきます。

### 悪い指示の例

```
@Claude なんか資料作って
```

これでは、Claudeも困ってしまいます。何の資料を、誰向けに、どんな形式で作ればいいのか分かりません。人間の部下に「なんか資料作って」と言ったら、困惑した顔をされますよね。AIも同じです。

### 良い指示の例

```
@Claude 来週火曜の営業会議で使うプレゼン資料の構成案を作ってください。

背景：
- 新規サービス「AIコンサルティング」の提案
- ターゲットは中小企業の経営者
- 会議の持ち時間は15分

構成に含めてほしいこと：
- 市場の課題（3つ程度）
- 当社のサービス概要
- 料金プラン（3段階）
- 導入事例（2社）
- 次のステップ

形式：
- スライド10枚程度の構成案
- 各スライドのタイトルと要点（箇条書き）
```

この2つの違いは一目瞭然です。良い指示には、次の3つの要素が含まれています。**「具体性」「ゴールの明示」「形式の指定」**です。


:::message
**プロンプトの書き方3原則**

AIへの指示（これを「プロンプト」と呼びます）を上手に書くための原則を3つ紹介します。

### 原則1：具体的に伝える

「いい感じの」「うまいこと」「適当に」は禁句です。

- ✕「いい感じのメールを書いて」
- ○「取引先の佐藤部長に送る、プロジェクト完了報告のメールを書いて。丁寧語で、300文字以内で」

何を（What）、誰に（Who）、どのくらい（How much）を明確にしてください。

### 原則2：ゴールを示す

エージェントに「何をしてほしいか」だけでなく、「何のためにそれが必要なのか」を伝えると、結果の質がぐっと上がります。

- ✕「競合3社の情報をまとめて」
- ○「来月の価格改定の参考にするため、競合3社の料金体系を比較表にまとめて。当社の現行価格との差額も入れて」

目的を伝えることで、AIは「何を重点的に調べればいいか」を自分で判断できるようになります。

### 原則3：形式を指定する

結果の「見た目」を指定しておくと、あとから加工する手間が省けます。

- ✕「調べた結果を教えて」
- ○「調べた結果を、以下の形式で表にまとめて。列：会社名 / サービス名 / 月額料金 / 特徴（50字以内）」

表、箇条書き、番号付きリスト、マークダウン形式。どんな形で返してほしいかを指定すれば、AIはその通りに整形してくれます。

この3つの原則は、本書の全ハンズオンを通じて一貫して使います。最初は少し面倒に感じるかもしれません。ただ、慣れれば自然に書けるようになりますし、具体的な指示を書くスキルは、人間の部下や外注先に仕事を頼むときにもそのまま活きます。
:::


## 作業系のタスクを体験する：ファイル作成

せっかくなので、チャット応答ではなくClaude Codeセッションが起動する「作業系」のタスクも試してみます。

**`#ai-secretary` チャンネルで、以下のメッセージを送信してください：**

```
@Claude 以下の内容で、自己紹介用のHTMLページを作成してください。

名前：山田太郎
肩書き：Webデザイナー / フリーランス
経験年数：5年
得意分野：コーポレートサイト、ECサイト
連絡先：taro@example.com

デザイン：
- シンプルでモダンなデザイン
- 背景は白、アクセントカラーは青（#2563EB）
- レスポンシブ対応（スマホでも見られるように）
```

今度は、Claudeが「これはファイルを作成する作業だ」と判断し、Claude Codeセッションが起動するはずです。

スレッドに進捗メッセージが表示され、しばらくすると完了の報告が返ってきます。「View Session」をクリックすると、ブラウザでClaude Code on the Webが開き、作成されたHTMLファイルの内容を確認できます。

HTMLファイルの中身が分からなくても問題ありません。重要なのは、**あなたが日本語で「こういうページを作って」と頼んだだけで、AIがプログラムコード（HTML）を自動生成してくれた**ということです。これが、エンジニアでなくてもAIエージェントが使える理由です。


## この章のまとめ

この章で体験したことを振り返ります。

1. **@Claude で話しかけるだけで、AIエージェントが動く**
   - 特別なコマンドや操作は不要
   - 日本語で自然に指示を出せる

2. **スレッドで追加指示ができる**
   - 文脈を覚えているので、修正や追加の依頼がスムーズ
   - 何度でもやり取りを重ねられる

3. **チャット応答とClaude Codeセッションの2種類がある**
   - テキスト生成はチャットで即座に返る
   - ファイル操作やコード実行はセッションが起動する

4. **良い指示 ＝ 具体的・ゴール明確・形式指定**
   - この3原則が、AIエージェントの実力を最大限に引き出す

AIエージェントとの会話は、慣れれば慣れるほどスムーズになります。最初の数回は「どう指示すればいいかな」と迷うかもしれませんが、使い続けるうちに、まるで長年一緒に働いてきた同僚に話すような自然さで指示を出せるようになります。

次の章からは、いよいよ5人のAIエージェントを「各部署に配属」していきます。最初に配属するのは、経営者の右腕であるAI秘書です。
