前章では、Managed Agentsの全体像を4つのコアコンセプト（Agent・Environment・Session・Events）で理解しました。「なるほど、仕組みはわかった。でも、どうやって始めるの？」——そう思っている方も多いだろう。

この章では手を動かす。Anthropic Consoleでのアカウント作成からAPIキーの取得、CLIツール「ant」のインストール、そしてはじめてのエージェント作成まで、すべて画面操作レベルで解説する。ターミナル未経験の方も心配無用だ。「コピペすれば動く」がこの章のモットー。

## 3-1. Anthropic Consoleでアカウントを作成する

まずは、Managed Agentsを使うための「入り口」を開きましょう。Anthropic Consoleは、APIキーの管理やエージェントの作成・監視を行うためのWebダッシュボードです。普段使い慣れたChatGPTの画面とは別のサービスですので、新規にアカウントを作成する必要があります。

### アカウント登録の手順

**ステップ1：Consoleにアクセスする**

ブラウザで以下のURLを開いてください。

```
https://console.anthropic.com
```

「Sign Up」ボタンをクリックします。Googleアカウントでのログイン、またはメールアドレスでの登録が選べます。普段使っているGoogleアカウントがあれば、それが一番手軽です。

**ステップ2：電話番号の認証**

セキュリティのため、SMS認証が必要です。携帯電話の番号を入力すると、6桁の認証コードが届きます。このコードを画面に入力すれば認証完了です。

> **つまずきポイント**: IP電話やVoIP番号（050で始まる番号など）は認証に使えないことがあります。携帯電話（090/080/070で始まる番号）を使用してください。

**ステップ3：組織（Organization）の作成**

初回ログイン時に、組織名を入力する画面が表示されます。個人利用であれば、自分の名前やハンドルネームで構いません。この組織名は後から変更できますので、深く考えなくて大丈夫です。

ここまでで、Consoleのダッシュボード画面にたどり着きます。所要時間は約3分です。

### APIキーの取得と安全な管理方法

アカウントが作成できたら、次はAPIキーを取得します。APIキーとは、「あなた本人であること」をシステムに証明するためのパスワードのようなものです。エージェントを動かすとき、このキーが必ず必要になります。

**APIキーの作成手順：**

1. Consoleの左メニューから「**API Keys**」をクリック
2. 画面右上の「**Create Key**」ボタンをクリック
3. キーの名前を入力（例：`my-first-agent`）
4. 「**Create Key**」をクリック

すると、`sk-ant-` で始まる長い文字列が表示されます。これがあなたのAPIキーです。

**ここで超重要な注意点があります。** このキーは、この画面を閉じると**二度と表示されません**。必ずこの場でコピーして、安全な場所に保存してください。コピーし忘れた場合は、このキーを削除して新しいキーを作り直す必要があります。

**APIキーの安全な保管方法：**

- パスワード管理アプリ（1Password、Bitwarden等）に保存する — **最も推奨**
- PCのメモ帳に保存して、ファイル名を「APIキー」などわかりやすくする — **次善策**
- 紙に書いて保管する — **アナログだが確実**

**やってはいけないこと：**

- SNSやブログにAPIキーを載せる（悪用されて高額請求の恐れ）
- メールやチャットでAPIキーを送る（流出リスク）
- GitHubなどのコード共有サービスにAPIキーを含むファイルをアップする

APIキーが流出すると、第三者があなたのアカウントでAPIを使い放題になり、その料金はすべてあなたに請求されます。クレジットカードの番号と同じくらい慎重に扱ってください。

### 支払い設定と利用上限の設定

APIキーを取得しただけでは、まだエージェントは動かせません。支払い方法の設定が必要です。

**支払い設定の手順：**

1. 左メニューの「**Settings**」→「**Billing**」をクリック
2. 「**Add Payment Method**」でクレジットカード情報を入力
3. 初期チャージ額を設定（最低$5〜。まずは$5で十分です）

**利用上限（Spending Limit）の設定：**

「使いすぎが怖い」という方は、利用上限を設定しましょう。

1. 「**Settings**」→「**Limits**」をクリック
2. 「**Set spend limit**」で月額の上限を設定

たとえば、月額$20（約3,000円）に設定しておけば、それ以上は課金されません。上限に達するとAPIがエラーを返すようになりますが、いつでもConsoleから上限を引き上げられます。本書のハンズオンをすべてこなしても、月額$10〜20程度に収まるはずですので、まずは$20に設定しておくことをおすすめします。

> **つまずきポイント**: 新規アカウントはBuild Tier 1（最初のランク）からスタートします。1分あたりのリクエスト数に制限がありますが、本書の内容であれば問題になることはありません。利用額が増えるにつれて自動的にTierが上がり、制限が緩和されます。

## 3-2. CLIツール「ant」をインストールする

Console画面でもエージェントは作れますが、本格的に使いこなすにはCLIツール「ant」が欠かせません。CLIとは「Command Line Interface」の略で、キーボードでコマンド（命令文）を入力してコンピュータを操作する方法のことです。

「え、黒い画面に文字を打つやつ？　無理……」と思った方、安心してほしい。この本では、すべてのコマンドをコピペで実行できるように書いている。自分で考えて入力する必要はほとんどない。

### ターミナル（コマンドプロンプト）の基本操作ガイド

「ant」をインストールする前に、まずターミナルの開き方を確認しましょう。

**Macの場合：**
1. 「Spotlight検索」（画面右上の虫眼鏡アイコン、またはCommand + スペース）を開く
2. 「terminal」と入力してEnter
3. 白い（または黒い）ウィンドウが開けばOK

**Windowsの場合：**
1. スタートメニューを開き、「PowerShell」と入力
2. 「Windows PowerShell」をクリック
3. 青いウィンドウが開けばOK

**Linuxの場合：**
1. デスクトップでCtrl + Alt + Tを押す
2. ターミナルが開けばOK

ターミナルが開いたら、試しに何か文字を入力してEnterキーを押してみてください。エラーメッセージが出ますが、壊れたわけではありません。「そのコマンドは知りません」と言っているだけです。ターミナルは、正しいコマンドだけに反応する仕組みです。

### Windows / Mac / Linux それぞれのインストール手順

それでは、antをインストールしましょう。OSごとに手順が異なります。

**Macの場合（Homebrew経由）：**

ターミナルで以下のコマンドをコピペして実行してください。

```bash
brew install anthropics/tap/ant
```

> **つまずきポイント**: 「brew: command not found」と表示された場合、Homebrew（Mac用のパッケージ管理ツール）がインストールされていません。以下のコマンドを先に実行してください。
> ```bash
> /bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"
> ```
> インストール完了後、ターミナルを再起動してから `brew install anthropics/tap/ant` を再実行してください。

**Windowsの場合：**

PowerShellで以下のコマンドをコピペして実行してください。

```powershell
winget install Anthropic.ant
```

> **つまずきポイント**: Windows 10の場合、wingetが入っていないことがあります。Microsoft Storeから「アプリ インストーラー」を検索してインストールしてください。Windows 11であれば最初から使えます。上記コマンドでうまくいかない場合は、Anthropic公式ドキュメント（platform.claude.com/docs）で最新のインストール手順を確認してください。

**Linuxの場合（Homebrew経由）：**

Linuxでも Homebrew が使えます。ターミナルで以下を実行してください。

```bash
brew install anthropics/tap/ant
```

Homebrewが未インストールの場合は、Macと同じコマンドで先にHomebrewをインストールしてください。

### 動作確認：`ant --version` で疎通チェック

インストールが完了したら、正しくインストールされたか確認しましょう。ターミナルに以下を入力してEnterを押してください。

```bash
ant --version
```

インストールされたバージョン番号が表示されれば成功です。

次に、APIキーを設定します。先ほどコピーしておいたAPIキーを使います。

```bash
export ANTHROPIC_API_KEY="sk-ant-ここにあなたのAPIキーを貼り付け"
```

> **つまずきポイント**: このコマンドは、ターミナルを閉じるとリセットされます。毎回入力するのが面倒な場合は、シェルの設定ファイル（Macなら `~/.zshrc`、Linuxなら `~/.bashrc`）にこの行を追加してください。やり方がわからなくても問題ない——本書のハンズオンではその都度APIキーの設定手順を記載している。

## 3-3. はじめてのエージェント作成 — Console画面で3ステップ

APIキーの取得とCLIのインストールが完了しました。いよいよ、はじめてのエージェントを作成しましょう。前章で学んだ4つのコンセプトのうち、ここではAgent（何をするか）、Environment（どこで動くか）、Session（実行する）の3つを、実際に画面で操作していきます。

### Agent定義の作成（GUI操作）

**ステップ1：Consoleでエージェントを作成する**

1. Console（https://console.anthropic.com）にログイン
2. 左メニューの「**Agents**」をクリック
3. 「**Create Agent**」ボタンをクリック

**ステップ2：エージェントの設定を入力する**

以下の3つを入力します。

- **Name**（名前）：`my-first-agent`
- **Model**（使用するAIモデル）：`claude-sonnet-4-6`（コスパが良い標準モデル）
- **System Prompt**（エージェントへの指示）：

```
あなたは親切なアシスタントです。
ユーザーの質問に日本語で丁寧に回答してください。
回答は簡潔に、3文以内でまとめてください。
```

4. 「**Save**」をクリック

これで、Agent定義が完成しました。画面に表示される`agent_id`（`agent_`で始まる文字列）をメモしておいてください。後で使います。

### Environment（実行環境）の作成

次に、エージェントが動く「場所」を作ります。

1. 左メニューの「**Environments**」をクリック
2. 「**Create Environment**」をクリック
3. 以下を設定：
   - **Name**：`default-env`
   - **ネットワーク設定**：「Internet Access」をONにする（Webツールを使うため）
4. 「**Save**」をクリック

表示される`environment_id`（`env_`で始まる文字列）もメモしておきましょう。

> **つまずきポイント**: Environmentはエージェントとは独立して作成します。1つのEnvironmentを複数のエージェントで使い回せます。「実行場所のテンプレート」だと思ってください。

### Session（タスク実行）の開始

最後に、AgentとEnvironmentを紐づけて、実際にエージェントを動かします。

1. 左メニューの「**Sessions**」をクリック
2. 「**Create Session**」をクリック
3. 以下を設定：
   - **Agent**：先ほど作成した `my-first-agent` を選択
   - **Environment**：先ほど作成した `default-env` を選択
   - **Message**：`こんにちは！あなたは何ができますか？`
4. 「**Start**」をクリック

数秒待つと、エージェントからの応答が画面に表示されます。「私は親切なアシスタントです。ご質問に日本語で丁寧にお答えできます。どうぞお気軽にお聞きください。」——このような回答が返ってくれば成功です。

おめでとうございます。あなたは今、はじめてのAIエージェントを動かしました。ChatGPTに質問するのとは違い、このエージェントはAPIを通じて動いています。つまり、このあと自動化の「仕組み」に組み込める、ということです。



### ハンズオン3：「自己紹介エージェント」をConsoleで作って動かす

**ゴール**: Console画面で独自のエージェントを作成し、自己紹介をさせる

**準備**: Anthropic Consoleにログイン済み、支払い設定済み

3-3で作成したAgent・Environmentをベースに、新しいSystem Promptを設定して、エージェントの振る舞いを変えてみましょう。

1. Consoleの「Agents」→「Create Agent」を開く
2. 以下を設定する：
   - **Name**: `self-intro-agent`
   - **Model**: `claude-sonnet-4-6`
   - **System Prompt**:
     ```
     あなたは自己紹介の達人です。
     ユーザーが名前と職業を伝えると、その人になりきって
     魅力的な自己紹介文を3パターン作成してください。
     それぞれ「カジュアル版」「ビジネス版」「SNS用」と
     ラベルをつけてください。
     ```
3. 「Save」をクリック
4. 「Environments」→「Create Environment」で `handson-env` を作成（ネットワーク設定はデフォルトでOK）
5. 「Sessions」→「Create Session」で以下を入力：
   - Agent: `self-intro-agent`
   - Environment: `handson-env`
   - Message: `田中太郎です。副業でWebライターをしています。`
6. 「Start」をクリックして、応答を確認する

**確認ポイント**: 3パターン（カジュアル版・ビジネス版・SNS用）の自己紹介文が生成されていればOKです。

> **つまずきポイント**: 応答が英語で返ってくる場合は、System Promptに「必ず日本語で回答してください」を追加してください。また、応答が途中で切れる場合は、モデルの出力トークン数の上限に達している可能性があります。System Promptを短くするか、「簡潔に」という指示を加えてみてください。



### ハンズオン4：CLIツール「ant」でエージェントを起動する（コピペ実行）

**ゴール**: ターミナルからantコマンドでエージェントを操作する

**準備**: antがインストール済み、APIキーが設定済み

1. ターミナルを開き、APIキーを設定する（まだ設定していない場合）：
   ```bash
   export ANTHROPIC_API_KEY="sk-ant-あなたのAPIキーをここに貼り付け"
   ```

2. ハンズオン3で作成したエージェントのagent_idとenvironment_idを確認する。Console画面で確認できます。以下のコマンドでも一覧表示できます：
   ```bash
   ant agents list
   ```

3. セッションを作成してエージェントを起動する（agent_idとenvironment_idは自分のものに置き換えてください）：
   ```bash
   ant sessions create \
     --agent-id agent_xxxxxxxx \
     --environment-id env_xxxxxxxx \
     --message "こんにちは！今日の東京の天気を教えてください"
   ```

4. エージェントの応答がターミナルに表示されます。セッションの状態を確認するには：
   ```bash
   ant sessions get --session-id session_xxxxxxxx
   ```

**確認ポイント**: ターミナルにエージェントからの応答がJSON形式で表示されればOKです。Console画面の「Sessions」にも同じセッションが表示されていることを確認してみてください。

> **つまずきポイント**: 「Error: ANTHROPIC_API_KEY not set」と表示された場合、APIキーの設定が抜けています。手順1のexportコマンドを再実行してください。また、「agent not found」エラーが出る場合は、agent_idのコピーミスが原因です。Console画面で正しいIDを確認し直してください。

**ターミナルが怖い人へ — 最低限これだけ覚えればOKの5コマンド**


「黒い画面」に苦手意識がある方は少なくありません。でも、本書で使うコマンドは実はほんの数種類です。以下の5つだけ覚えておけば、この本のすべてのハンズオンをこなせます。

**1. `cd`（移動する）**

「Change Directory」の略。フォルダを移動するコマンドです。

```bash
cd Desktop          # デスクトップに移動
cd ..               # 1つ上のフォルダに戻る
```

**2. `ls`（中身を見る）**（Windowsは `dir`）

今いるフォルダの中にあるファイル一覧を表示します。

```bash
ls                  # ファイル一覧を表示
```

**3. `cat`（ファイルを読む）**（Windowsは `type`）

ファイルの中身をターミナルに表示します。

```bash
cat result.txt      # result.txtの中身を表示
```

**4. `export`（設定する）**（Windowsは `$env:変数名 = "値"`）

環境変数（APIキーなど）を設定します。

```bash
export ANTHROPIC_API_KEY="sk-ant-xxxxx"    # Mac/Linux
$env:ANTHROPIC_API_KEY = "sk-ant-xxxxx"    # Windows PowerShell
```

**5. `ant`（エージェントを操作する）**

本書の主役。エージェントの作成・実行・管理をすべてこのコマンドで行います。

```bash
ant agents list                # エージェント一覧
ant sessions create ...        # セッション作成
ant sessions get ...           # セッション状態確認
```

どのコマンドも、「打ち間違えたらPCが壊れる」ということはありません。最悪の場合でも「そんなコマンドはありません」とエラーが出るだけです。怖がらずに、まずはコピペから始めてみてください。慣れてくると、マウスで画面をクリックするより速く操作できるようになります。

## まとめ

- **Anthropic Consoleでアカウントを作成し、APIキーを取得する**。Console（https://console.anthropic.com）で登録→SMS認証→APIキー作成の3ステップ。APIキーは表示された瞬間にコピーして安全な場所に保管する
- **支払い設定と利用上限を忘れずに**。初期チャージ$5＋月額上限$20で始めれば、本書のハンズオンは余裕でまかなえる。上限を設定しておけば想定外の課金を防げる
- **CLIツール「ant」はコピペ1行でインストールできる**。Macは`brew install anthropics/tap/ant`、Windowsは`winget install Anthropic.ant`。`ant --version`で動作確認
- **エージェント作成は3ステップ**。Agent（何をするか）→ Environment（どこで動くか）→ Session（実行する）。Console画面でもCLIでも同じ流れ
- **ターミナルは怖くない**。`cd`, `ls`, `cat`, `export`, `ant`の5コマンドで本書は乗り切れる。間違えてもPCは壊れない

次の第4章では、エージェントの「頭脳」にあたるシステムプロンプトの設計技術を学びます。「何をさせるか」を言語化し、エージェントに的確に伝える方法——これがエージェント活用の成果を左右する最も重要なスキルです。今回のセットアップで「動かす環境」は整いました。次は「上手に動かす方法」を身につけていきましょう。
