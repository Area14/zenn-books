前章では、ライティング副業の自動化を扱った。ブログ記事や電子書籍の執筆をエージェントに任せる方法を学んだが、「書く」だけが副業ではない。ライティングと並んで需要が高いのが「調べてまとめる」仕事——リサーチ・レポート副業だ。

クラウドソーシングサイトを覗いてみてほしい。「○○業界の市場調査をお願いします」「競合5社の比較レポートを作成してほしい」「アンケートデータを集計して報告書にまとめてください」——こうした案件が毎日投稿されている。クラウドワークスやランサーズでのリサーチ系案件の単価は1件あたり5,000円〜50,000円。専門性が高い調査レポートなら10万円を超える案件も珍しくない。

しかし、手作業でリサーチレポートを書くのは時間がかかる。情報を検索し、複数のソースを読み比べ、データを整理し、構造化されたレポートにまとめる。1本のレポートに5〜10時間かかることもざらだ。これこそ、エージェントの「判断・実行・繰り返し」の力が最も活きる領域である。

この章では、市場調査レポートエージェント、競合分析エージェント、データ整理・分析エージェントの3つを構築し、リサーチ・レポート副業を自動化する方法を示す。

## 6-1. 市場調査レポートエージェントを構築する

「○○市場の現状と今後の展望をレポートにまとめてほしい」——こうした依頼を受けたとき、あなたはどう進めますか？ まずGoogleで検索し、業界レポートや統計データを探し、複数の記事を読み込み、情報を整理して、Wordやスプレッドシートにまとめる。この一連の作業を、エージェントに丸ごと任せることができます。

### Web searchとWeb fetchを組み合わせた情報収集

市場調査エージェントの核心は、**Web search**と**Web fetch**の2つのツールの使い分けにあります。

**Web search**は、キーワードで情報を広く検索するツールです。「ペットフード市場 2026年 市場規模」のようなクエリを投げると、関連する検索結果が返ってきます。ただし、検索結果はタイトルとスニペット（概要文）だけなので、詳細な情報までは得られません。

そこで活躍するのが**Web fetch**です。Web searchで見つけた有望なURLを指定して、ページの全文を取得します。公式の統計データや業界レポートの詳細な数字を読み取るには、このWeb fetchが不可欠です。最新バージョンのWeb fetchでは動的フィルタリング機能が搭載されており、ページ全体を読み込むのではなく、必要な情報だけを抽出してコンテキストに取り込みます。これにより、トークン消費を抑えながら効率的に情報収集ができます。

エージェントのシステムプロンプトでは、この2つのツールを段階的に使うよう指示します。

**ステップ1：Web searchで全体像を把握する**
- 対象市場のキーワードで3〜5回検索する
- 検索クエリを変えながら、異なる角度から情報を集める
- 「市場規模」「成長率」「主要プレイヤー」「課題」など、レポートの各セクションに対応するクエリを設計する

**ステップ2：Web fetchで詳細データを取得する**
- 検索結果から信頼性の高いソース（政府統計、業界団体、調査会社のレポート）を選ぶ
- 該当ページをWeb fetchで取得し、具体的な数字やデータを抽出する
- 最低でも3つ以上の異なるソースからデータを取得する

この「検索→詳細取得」の2段構えが、エージェントによるリサーチの精度を大きく左右します。

### 収集データの構造化とMarkdownレポート出力

情報を集めるだけでは、レポートにはなりません。収集したデータを構造化し、読みやすいレポートにまとめる——ここがエージェントの腕の見せどころです。

システムプロンプトで「出力形式」を明確に指定することで、エージェントは毎回同じフォーマットのレポートを生成します。Markdownで出力させるのがおすすめです。理由は3つあります。

1. **Markdownはシンプルで読みやすい**。見出し、箇条書き、表がテキストだけで表現できます
2. **変換が容易**。MarkdownからWord、PDF、HTMLへの変換ツールが豊富です。納品形式に合わせて柔軟に対応できます
3. **差分管理しやすい**。テキストベースなので、前回レポートとの変更点がひと目でわかります

レポートの構造は、以下のようなテンプレートで定義します（章末のテンプレートを参照）。

```
# {調査テーマ} 市場調査レポート
## エグゼクティブサマリー（300字以内）
## 1. 市場概要
## 2. 市場規模と成長率
## 3. 主要プレイヤー
## 4. トレンドと機会
## 5. リスクと課題
## 6. 結論と提言
## 出典一覧
```

エージェントに「この構造に従ってレポートを作成してください」と指示するだけで、毎回一貫したフォーマットの成果物が得られます。手作業でゼロから構成を考える必要はありません。

### 複数ソースのクロスチェック設計

リサーチレポートの信頼性を左右するのが、情報の裏付けです。1つのソースだけに頼ると、その情報が誤っていた場合にレポート全体の信頼性が崩壊します。エージェントにも「クロスチェック」の習慣を組み込みましょう。

システムプロンプトに以下のルールを記述します。

- **「重要な数字（市場規模・成長率・シェアなど）は、2つ以上の独立したソースで確認する」**
- **「ソースが1つしか見つからない場合は、レポートに『※単一ソース』と注記する」**
- **「情報の出典を必ず記載する（URL・発行元・発行日）」**

このルールを入れるだけで、エージェントは自動的に複数の検索を実行し、情報を照合するようになります。第2章で解説した「判断する力」が、ここで活きるわけです。情報が食い違っている場合は、「ソースAでは○○、ソースBでは△△と報告されており、差異がある」とレポートに明記してくれます。

人間のリサーチャーなら当たり前にやることですが、これをシステムプロンプトに書くだけで自動化できるのがエージェントの強みです。

## 6-2. 競合分析エージェントの設計

次に、もう一つの人気案件ジャンルである「競合分析」のエージェントを設計しましょう。「自社の競合5社について、サービス内容・価格・強み弱みを比較レポートにまとめてほしい」——こうした依頼は、マーケティング支援の副業で頻繁に発生します。

### 競合サイトの定期巡回と変更検知

競合分析の基本は、対象企業のWebサイトやSNSを定期的にチェックし、変化を捉えることです。新サービスの発表、価格改定、キャンペーンの実施——こうした変化は、競合の戦略を読み解くヒントになります。

エージェントを使えば、この「定期チェック」を自動化できます。具体的な設計はこうです。

**情報ソースの定義：** まず、分析対象の競合企業のURL一覧をテキストファイルで準備します。たとえば `competitors.txt` に以下のように記載します。

```
# 競合企業リスト
https://example-company-a.com/news
https://example-company-b.com/press-release
https://example-company-c.com/blog
```

エージェントは、Web fetchでこれらのURLを順番に取得し、前回取得時との差分を抽出します。具体的には、Writeツールで前回の取得結果をファイルに保存しておき、次回の実行時にReadツールで前回データを読み込み、Bashツールで `diff` コマンドを使って差分を検出する——という流れです。

**変更検知のフロー：**

1. `competitors.txt` から対象URLを読み込む（Readツール）
2. 各URLの現在のコンテンツを取得する（Web fetchツール）
3. 前回保存データと比較する（Bashツールで `diff`）
4. 変更があった箇所を抽出し、レポートに追記する（Writeツール）
5. 今回のデータを次回用に保存する（Writeツール）

この仕組みを週1回のペースで実行すれば、「先週から今週にかけて、競合A社が新プランを発表しました」といった変化を自動的にキャッチできます。

### 分析フレームワーク（SWOT等）に沿った自動レポート

競合分析レポートの品質を安定させるコツは、分析フレームワークを使うことだ。代表的なのがSWOT分析——Strengths（強み）・Weaknesses（弱み）・Opportunities（機会）・Threats（脅威）の4つの観点で対象を評価する手法である。

エージェントにSWOT分析を自動化させるには、システムプロンプトにフレームワークの「枠組み」を埋め込みます。

```
## 競合分析レポートの出力形式

各競合企業について、以下のSWOTフレームワークで分析してください。

### {企業名}
| 観点 | 分析内容 |
|------|---------|
| **Strengths（強み）** | ・（3項目以上） |
| **Weaknesses（弱み）** | ・（3項目以上） |
| **Opportunities（機会）** | ・（2項目以上） |
| **Threats（脅威）** | ・（2項目以上） |

分析の根拠として、各項目に具体的な事実・データを付記すること。
```

このように出力テンプレートを指定すると、エージェントはWeb searchで各企業の情報を収集し、フレームワークの各項目を埋めていきます。5社分のSWOT分析をまとめても、エージェントなら20〜30分で完了します。手作業なら丸1日かかる作業です。

SWOT以外にも、**3C分析**（Customer：顧客、Company：自社、Competitor：競合）や**ポーターの5フォース分析**（業界の競争構造を5つの力で分析するフレームワーク）など、目的に応じてフレームワークを切り替えられます。テンプレートをいくつか用意しておけば、クライアントの要望に合わせて柔軟に対応できる。

## 6-3. データ整理・分析エージェント

リサーチ副業では、「すでにあるデータを整理・分析してほしい」という依頼も多くあります。たとえば、「アンケート結果のCSVデータを集計して、グラフ用のデータと考察をレポートにまとめてほしい」といった案件です。

### CSVデータの読み込みと集計（Bashツール活用）

ここで威力を発揮するのが**Bashツール**です。第2章で解説した通り、BashツールはLinuxコンテナ内でコマンドやスクリプトを実行できます。「自分はプログラミングできない」という方も心配は要らない。エージェントが自分でPythonコードを書いて実行してくれる。

具体的な流れを見てみましょう。クライアントから「顧客満足度アンケート（500件）のCSVデータを分析してほしい」という依頼を受けたとします。

**ステップ1：データの準備**
クライアントから受け取ったCSVファイル（例：`survey_data.csv`）をエージェントの作業環境に配置します。Managed Agentsでは、Environmentのマウントファイル機能を使ってファイルを事前に配置できます。

**ステップ2：エージェントがデータを読み込む**
エージェントはReadツールでCSVファイルの先頭数行を確認し、データ構造（列名、データ型、件数）を把握します。

**ステップ3：Bashツールで集計スクリプトを実行**
エージェントは自分でPythonスクリプトを生成し、Bashツール経由で実行します。たとえば、こんなコードを自動で書いて動かします。

```python
import csv
from collections import Counter

# CSVファイルの読み込み
with open('survey_data.csv', 'r', encoding='utf-8') as f:
    reader = csv.DictReader(f)
    data = list(reader)

# 満足度スコアの集計
scores = [int(row['satisfaction_score']) for row in data]
avg_score = sum(scores) / len(scores)
score_dist = Counter(scores)

print(f"回答数: {len(data)}")
print(f"平均満足度: {avg_score:.1f}")
print(f"スコア分布: {dict(sorted(score_dist.items()))}")
```

あなたがPythonを書く必要はありません。エージェントが「CSVを読み込んで、満足度スコアの平均と分布を集計してください」という指示を受けて、自分でコードを生成・実行します。結果が間違っていれば、エージェントは自分でコードを修正して再実行する——これが「判断する力」です。

**ステップ4：分析結果をレポートにまとめる**
集計結果をもとに、Writeツールで分析レポートをMarkdownファイルとして出力します。

### グラフ・チャート用データの自動生成

「グラフも付けてほしい」という依頼は多いですが、エージェントはグラフの画像を直接生成するのは得意ではありません。その代わり、**グラフ作成用のデータ**を整形して出力する方法が実用的です。

たとえば、Googleスプレッドシートに貼り付ければすぐにグラフ化できるCSVデータや、Mermaid記法（テキストでグラフを表現するフォーマット）でのチャートデータを出力させることができます。

```
## 満足度スコア分布

| スコア | 回答数 | 割合 |
|--------|--------|------|
| 1（非常に不満） | 12 | 2.4% |
| 2（不満） | 38 | 7.6% |
| 3（普通） | 125 | 25.0% |
| 4（満足） | 210 | 42.0% |
| 5（非常に満足） | 115 | 23.0% |
```

このような表形式のデータをスプレッドシートに貼り付ければ、ワンクリックでグラフが作れます。クライアントへの納品時には「レポート本文（Markdown or Word）＋グラフ用データ（CSV）」のセットで納品すると、付加価値の高い成果物になります。

また、BashツールでPythonの `matplotlib` ライブラリを使えば、エージェントがグラフ画像を直接生成することも可能です。Environmentのプリインストールパッケージに `matplotlib` を追加しておけば、棒グラフや円グラフをPNG画像として出力できます。

## 6-4. コスト試算：月5本の調査レポート案件を自動化する場合

「エージェントを使えば効率的なのはわかった。でも、実際にいくら稼げて、いくらかかるの？」——副業として取り組むなら、ここが最も気になるポイントだ。具体的な数字で試算する。

### 案件単価 vs エージェント運用コスト

クラウドソーシングサイトでのリサーチ・レポート案件の相場は以下の通りです。

| 案件タイプ | 単価相場 |
|-----------|---------|
| 簡易な市場調査レポート（2,000〜3,000字） | 5,000〜10,000円 |
| 競合分析レポート（5社程度） | 10,000〜30,000円 |
| 本格的な業界調査レポート（10,000字以上） | 30,000〜100,000円 |
| データ集計・分析レポート | 10,000〜50,000円 |

ここでは「月5本の市場調査レポート（単価15,000円/本）を受注する」ケースで試算します。

**収入側：**
- 月5本 × 15,000円 = **75,000円/月**

**エージェント運用コスト：**
- レポート1本あたりのエージェント稼働時間：約15〜20分
  （Web search × 5〜8回 + Web fetch × 3〜5回 + レポート生成 + クロスチェック）
- 月5本の合計稼働時間：約75〜100分（約1.3〜1.7時間）
- ランタイム料金：約1.7時間 × $0.08/時間 = $0.14（約21円）
- API利用料（トークン消費）：約$3〜5（約450〜750円）
- **合計コスト：月額約471〜771円**

**利益：**
- 75,000円 − 約771円 = **約74,200円/月**
- **利益率：約99%**（74,229円÷75,000円×100≒99%）

ChatGPT Plusの月額約3,000円よりもはるかに安い運用コストで、月7万円以上の利益が出る計算です。ランタイム料金はわずか約21円で、コストの大部分はAPI利用料（トークン課金）が占めます。

### 手動作業時間の削減効果

コストだけでなく、時間の削減効果も見てみましょう。

| 項目 | 手作業 | エージェント活用 |
|------|--------|----------------|
| 情報収集 | 2〜3時間/本 | 自動（約10分） |
| データ整理・構造化 | 1〜2時間/本 | 自動（約5分） |
| レポート執筆 | 2〜3時間/本 | 自動（約5分） |
| 品質チェック・修正 | 0.5〜1時間/本 | 30分/本（人間が確認） |
| **合計（1本あたり）** | **5.5〜9時間** | **約50分** |
| **月5本の合計** | **27.5〜45時間** | **約4時間10分** |

手作業だと月27〜45時間かかるところが、エージェント活用なら約4時間に短縮されます。**削減率は約85〜90%**です。ただし、「品質チェック・修正」の30分は人間が行う想定です。エージェントの出力をそのまま納品するのではなく、必ず目を通して事実確認と文体の調整を行ってください。これが品質を保つための最低限のラインです。

浮いた時間で案件数を増やすこともできますし、本業や家族との時間に充てることもできます。副業の自動化は「もっと稼ぐ」だけでなく「時間を取り戻す」効果もあるのです。



### ハンズオン8：市場調査レポートエージェントをつくる（テンプレート付き）

**ゴール**: 指定したテーマの市場調査レポートを自動生成するエージェントを構築する

**準備**: Anthropic Consoleにログイン済み、APIキー設定済み、antインストール済み

1. Consoleの「Agents」→「Create Agent」で新しいエージェントを作成します。以下を設定してください。
   - **Name**: `market-research-agent`
   - **Model**: `claude-sonnet-4-6`
   - **Tools**: `agent_toolset_20260401`（Web search、Web fetch、Read、Write、Bashを含む標準ツールセット）
   - **System Prompt**: 章末の【テンプレート】をコピペしてください

2. 「Environments」→「Create Environment」で実行環境を作成します。
   - **Name**: `research-env`
   - **ネットワーク設定**: 「Internet Access」をONにする（Web searchとWeb fetchを使うため）

3. 「Sessions」→「Create Session」でエージェントを起動します。以下のメッセージを送信してください。

   ```
   「日本のペットフード市場」について市場調査レポートを作成してください。
   レポートは指定のMarkdownテンプレートに従い、output/pet_food_market_report.md に保存してください。
   ```

4. エージェントが動き始めます。進行状況はEventsで確認できます。以下のような流れで処理が進みます。
   - Web searchで「ペットフード 市場規模 2026」「ペットフード 成長率」などを検索
   - Web fetchで統計データや業界レポートの詳細ページを取得
   - 収集した情報をMarkdownレポートとして構造化
   - ファイルに保存

5. 完了したら、CLIでレポートの内容を確認します。
   ```bash
   ant sessions get --session-id session_xxxxxxxx
   ```
   または、Console画面のSessionsから該当セッションをクリックして結果を確認してください。

**確認ポイント**: 生成されたレポートに以下が含まれていれば成功です。
- エグゼクティブサマリー
- 市場規模の具体的な数字（出典付き）
- 主要プレイヤー3社以上のリスト
- トレンド分析
- 出典一覧（3つ以上のソース）

> **つまずきポイント**: レポートの内容が薄い場合は、System Promptの「検索クエリの数」を増やしてみてください。「各セクションにつき最低2回のWeb searchを実行する」と指示を追加すると、情報量が改善します。また、Web fetchでエラーが出る場合は、対象URLがアクセス制限をかけている可能性があります。その場合はエージェントが自動的に別のソースを探しますが、うまくいかない場合は「政府統計や業界団体のサイトを優先的に参照する」と指示を追加してください。



### ハンズオン9：CSVデータを読み込んで分析レポートを生成するエージェント

**ゴール**: CSVデータを読み込み、集計・分析してMarkdownレポートを自動生成する

**準備**: Anthropic Consoleにログイン済み、サンプルCSVデータ（以下の手順で作成）

1. まず、テスト用のCSVデータを作成します。以下の内容をコピーして `sample_survey.csv` として保存してください（メモ帳やテキストエディタで作成可能です）。

   ```csv
   id,age_group,gender,satisfaction_score,recommend,comment
   1,30代,男性,4,はい,使いやすい
   2,40代,女性,5,はい,非常に満足
   3,20代,男性,3,いいえ,普通
   4,50代,女性,4,はい,概ね満足
   5,30代,女性,2,いいえ,改善してほしい
   6,40代,男性,5,はい,素晴らしい
   7,20代,女性,3,いいえ,可もなく不可もなく
   8,30代,男性,4,はい,良い
   9,50代,男性,1,いいえ,不満
   10,40代,女性,5,はい,最高
   ```

2. Consoleの「Agents」→「Create Agent」で新しいエージェントを作成します。
   - **Name**: `data-analyst-agent`
   - **Model**: `claude-sonnet-4-6`
   - **Tools**: `agent_toolset_20260401`
   - **System Prompt**:
     ```
     あなたはデータ分析の専門家です。以下の手順でCSVデータを分析してください。

     1. Readツールで指定されたCSVファイルを読み込む
     2. データの概要（行数・列数・列名）を把握する
     3. Bashツールを使い、Pythonスクリプトで以下を集計する：
        - 各列の基本統計（平均、最大、最小、分布）
        - カテゴリ別のクロス集計
        - 注目すべきパターンや傾向
     4. 分析結果をMarkdownレポートとして出力する
        - レポートにはデータの概要、主要な集計結果、考察を含める
        - 表形式でデータを見やすく整理する
        - グラフ化用のCSVデータも併せて出力する
     5. レポートを指定パスに保存する

     注意：分析は客観的なデータに基づくこと。推測には「推測」と明記する。
     ```

3. 「Environments」→「Create Environment」で実行環境を作成します。
   - **Name**: `analysis-env`
   - **ネットワーク設定**: OFF（データ分析にインターネットは不要）
   - **マウントファイル**: `sample_survey.csv` をアップロード

4. 「Sessions」→「Create Session」で以下のメッセージを送信します。

   ```
   sample_survey.csv を分析して、顧客満足度調査の分析レポートを作成してください。
   分析レポートは output/survey_analysis.md に保存してください。
   グラフ作成用のCSVデータも output/chart_data.csv に出力してください。
   ```

5. エージェントの処理が完了したら、生成されたファイルを確認します。

**確認ポイント**: 以下が含まれていれば成功です。
- データ概要（回答数、列情報）
- 満足度スコアの平均値と分布
- 年代別・性別のクロス集計
- 推奨意向（recommend）の集計
- 考察と提言
- グラフ用のCSVファイル

> **つまずきポイント**: Pythonスクリプトの実行でエラーが出る場合、CSVファイルの文字コードが原因であることが多いです。日本語を含むCSVは `encoding='utf-8'` で読み込む必要があります。エージェントは通常これを自動判断しますが、うまくいかない場合は「CSVの文字コードはUTF-8です」とメッセージに追記してください。また、Pythonが実行できない旨のエラーが出た場合は、Environmentの設定でPythonがプリインストールされているか確認してください（デフォルトでインストール済みです）。

## まとめ

- **市場調査レポートはWeb search＋Web fetchの2段構えで自動化できる**。検索で全体像を把握し、Web fetchで詳細データを取得する。クロスチェックのルールをシステムプロンプトに組み込むことで、信頼性の高いレポートがエージェント単体で完成する
- **競合分析はSWOTなどのフレームワークをテンプレート化する**。エージェントに枠組みを与えるだけで、5社分の分析が20〜30分で完了する。定期巡回と変更検知を組み合わせれば、継続的な競合モニタリングも自動化できる
- **データ分析はBashツール経由のPython実行が鍵**。CSVの読み込みから集計・分析まで、エージェントが自分でコードを書いて実行してくれる。プログラミングの知識は不要
- **月5本の調査レポートで月収約74,200円、運用コストはわずか約500〜800円**。利益率99%、手作業比で時間を85〜90%削減。浮いた時間で案件数を増やすことも、本業や家族との時間に充てることもできる
- **品質の最終チェックは必ず人間が行う**。エージェントの出力をそのまま納品せず、事実確認と文体調整に30分を使う。これが信頼されるリサーチャーであり続けるための最低ライン
