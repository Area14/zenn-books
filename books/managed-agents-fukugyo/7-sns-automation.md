前章では、リサーチ・レポート副業の自動化を扱った。「調べて、まとめて、納品する」という一連の流れをエージェントに任せる方法を身につけたが、実はこのスキルセットがそのまま活きる副業ジャンルがある。それが、SNS運用代行だ。

SNS運用代行の市場は拡大を続けている。企業がInstagramやXの運用を外部に委託するケースは年々増えており、フリーランスへの依頼相場は月額数万〜10万円、企業向けの代行会社なら月額20〜50万円が中心価格帯だ。投稿文の作成、ハッシュタグの選定、投稿スケジュールの管理、パフォーマンス分析——これらの作業の多くは「パターン化できる定型業務」、つまりエージェントが得意とする領域にそのまま当てはまる。

この章では、SNS運用の主要タスクを3つのエージェントで自動化する。投稿作成、コンテンツカレンダー生成、エンゲージメント分析——この3つを組み合わせれば、1アカウントあたり月10〜15時間かかっていた作業が2〜3時間に縮む。

## 7-1. SNS投稿作成エージェントを構築する

SNS運用で最も時間がかかるのが、投稿文の作成です。毎日〜週数回の投稿を維持するには、ネタ探し、文章の作成、ハッシュタグの選定を繰り返す必要があります。このセクションでは、X・Instagram・noteの3プラットフォームに対応した投稿作成エージェントを設計します。

### プラットフォーム別（X / Instagram / note）の投稿文生成

SNSのプラットフォームごとに、最適な投稿スタイルはまったく異なります。X（旧Twitter）の投稿上限は280文字ですが、日本語ではおおむね140字前後が読みやすい目安とされています。つまり短文勝負。Instagramはビジュアルが主役で、キャプションは補助的な役割。noteは長文のストーリーテリングが求められます。1つのエージェントで「全部同じ文章を生成」しても効果は出ません。

そこで、システムプロンプトにプラットフォーム別の制約を明記します。具体的には、以下の3つを指定します。

**1. 文字数制限**
- X：140字以内（日本語）。句読点を含めてカウント
- Instagram：キャプション2,200字以内。ただし最初の125字が「もっと見る」の前に表示されるため、冒頭に核心を書く（日本語の場合は全角文字の影響で約40〜50字程度で「もっと見る」が表示される場合があります）
- note：1,500〜3,000字程度。見出しと段落で構造化する

**2. トーンと文体**
- X：端的で、断言口調。「〜です」より「〜だ」「〜する」が好まれる
- Instagram：親しみやすく、共感を誘う語りかけ調。絵文字を適度に使用
- note：丁寧語ベース。体験談や具体例を交え、読者が「自分ごと」に感じる文章

**3. 構成パターン**
- X：主張→根拠→行動喚起（CTA）の3行構成
- Instagram：フック（冒頭の引き）→本文→ハッシュタグブロック
- note：導入→課題提起→解決策→まとめの4パート構成

エージェントのシステムプロンプトで、これらを「出力フォーマット」として定義しておけば、テーマを1行渡すだけでプラットフォームに最適化された投稿文が生成されます。具体的なプロンプト例は、章末の【テンプレート】で紹介します。

### トレンドキーワードの自動リサーチと反映

SNS投稿のリーチ（どれだけの人に届くか）を最大化するには、「今話題になっているキーワード」を投稿に盛り込むことが重要です。しかし、毎日トレンドを手動でチェックして投稿に反映するのは大変な手間です。

ここでWeb searchツールが活躍します。エージェントのシステムプロンプトに、以下のような指示を加えます。

```
投稿文を作成する前に、必ず以下の手順を実行してください：
1. Web searchで「{業界キーワード} トレンド 2026」を検索
2. 検索結果から、直近1週間で話題になっているトピックを3つ抽出
3. そのうち、クライアントのブランドイメージに合うものを1つ選択
4. 選択したトピックを自然に投稿文に組み込む
```

たとえば、フィットネスジムのSNS運用を代行している場合、エージェントは「フィットネス トレンド 2026」を検索し、「朝活ワークアウト」がトレンドだと判断したら、それを自然に取り入れた投稿文を作成します。「今朝5時に起きてジムに来た方、今週だけで3倍に増えました。朝活ワークアウト、試してみませんか？」——このような投稿が自動で生まれるわけです。

### 画像キャプション・ハッシュタグの自動生成

Instagramでは、ハッシュタグの選定が投稿のリーチを大きく左右します。一般的には、1投稿あたり5〜15個のハッシュタグが推奨されています。しかし、毎回適切なハッシュタグを調べるのは地味に時間がかかります。

エージェントには、ハッシュタグを3つのカテゴリーで生成させましょう。

- **ビッグタグ**（投稿数100万以上）：#フィットネス #筋トレ など。リーチは広いが競争が激しい。2〜3個
- **ミドルタグ**（投稿数1万〜100万）：#朝活ワークアウト #ジム初心者 など。適度なリーチと発見性。5〜7個
- **スモールタグ**（投稿数1万以下）：#渋谷ジム #パーソナル体験記 など。ニッチだが上位表示されやすい。3〜5個

この「ビッグ・ミドル・スモール」の比率をシステムプロンプトで指定しておけば、エージェントが投稿内容に応じたハッシュタグセットを自動生成します。Web searchで各ハッシュタグの投稿数を確認させることも可能です。

画像キャプションについても同様です。クライアントから提供された画像の説明テキスト（ALTテキスト）をエージェントに渡せば、その内容に沿ったキャプションを生成できます。「この画像はジムの内観写真です。明るい照明、最新のマシンが並んでいます」と伝えるだけで、「朝の静かなジムで、自分だけの時間を。最新マシンが揃った空間で、今日も一歩前へ」のようなキャプションが出てきます。

## 7-2. コンテンツカレンダー自動生成エージェント

投稿文を1つずつ作るだけでなく、「月間でどのような投稿をするか」という計画を立てることが、SNS運用の成果を大きく左右します。コンテンツカレンダーとは、いつ・何を・どのプラットフォームで投稿するかを一覧にしたスケジュール表のことです。

### 月間投稿計画の自動作成

コンテンツカレンダーの作成は、経験豊富なSNS担当者でも毎月2〜3時間はかかる作業です。エージェントに任せれば、これを5〜10分に短縮できます。

エージェントに渡す情報は、以下の4つです。

1. **クライアント情報**：業種、ターゲット層、ブランドのトーン
2. **投稿頻度**：週3回、毎日、など
3. **コンテンツの柱**：たとえば「商品紹介」「お客様の声」「業界ニュース」「裏側紹介」の4カテゴリー
4. **月間イベント**：セール、季節行事、業界イベントなど

これらの情報をシステムプロンプトに定義し、「2026年5月のコンテンツカレンダーを作成してください」と指示すれば、エージェントは以下のようなMarkdown形式のカレンダーを出力します。

```
| 日付 | 曜日 | プラットフォーム | カテゴリー | 投稿テーマ | ハッシュタグ案 |
|------|------|------------------|------------|------------|---------------|
| 5/1  | 木   | X                | 業界ニュース | GW中のフィットネストレンド | #GWトレーニング |
| 5/2  | 金   | Instagram        | 商品紹介   | 新プロテインバーの紹介 | #プロテインバー |
| 5/5  | 月   | note             | お客様の声 | 3か月で10kg減量した会員様インタビュー | - |
| ...  | ...  | ...              | ...        | ...        | ...           |
```

エージェントはWeb searchで月間イベント（祝日、記念日、業界イベント）を自動で調べ、カレンダーに反映します。「5月4日はスターウォーズの日だから、ユーモア投稿を1つ入れよう」といった提案もしてくれます。

### 曜日・時間帯別の最適投稿パターン設計

投稿するタイミングも成果を左右する重要な要素です。2025〜2026年の調査データによると、SNSの利用率は夜21〜22時にピークを迎えます。主要プラットフォーム別の推奨投稿時間帯は以下の通りです。

**X（旧Twitter）**
- 平日：朝6〜7時（通勤時間帯）、昼12〜13時（昼休み）、夜20〜22時（リラックスタイム）
- 木曜夜のエンゲージメントが最も高く、通常比で+150%以上というデータもあります
- 週末：土曜の午前中がねらい目

**Instagram**
- 平日：朝7〜9時、昼11〜13時、夜19〜21時
- リール（短尺動画）は20時以降の投稿で再生数が伸びやすい傾向
- 投稿後30分〜1時間のエンゲージメントがアルゴリズム評価に大きく影響

**note**
- 平日：朝7〜8時（通勤読書層）、夜21〜23時（じっくり読む層）
- 週末：土曜午前が安定して読まれやすい

エージェントのコンテンツカレンダーには、このデータに基づいた推奨投稿時間も含めましょう。システムプロンプトに上記のデータを組み込んでおけば、カレンダーの各投稿に「推奨時間：20:00」のように自動で付与されます。

ただし、これはあくまで一般的な傾向です。実際の最適時間はアカウントごとに異なるため、7-3で解説するエンゲージメント分析エージェントのデータを基に、随時調整していくことが重要です。

## 7-3. エンゲージメント分析エージェント

投稿を「作って終わり」ではもったいない。どの投稿が伸びて、どの投稿が伸びなかったのかを分析し、次の投稿に活かす——このPDCAサイクルを回すことが、SNS運用代行の価値を高めます。

### 投稿パフォーマンスデータの収集と分析

エンゲージメント分析エージェントは、クライアントのSNSアカウントの投稿データを整理・分析するエージェントです。データの取得方法には、大きく2つのアプローチがあります。

**アプローチ1：CSVデータを手動エクスポートして分析**

最もシンプルで確実な方法です。X、Instagramともに、管理画面（Xアナリティクス、Instagramインサイト）からCSV形式でデータをエクスポートできます。このCSVファイルをエージェントに渡して分析させます。

エージェントはBashツールを使ってCSVを読み込み、以下のような指標を自動計算します。

- **エンゲージメント率**：（いいね + コメント + リポスト） ÷ インプレッション × 100
- **投稿カテゴリー別の平均エンゲージメント率**：「商品紹介」と「お客様の声」ではどちらが反応がよいか
- **時間帯別のパフォーマンス**：何時に投稿した記事が最もエンゲージメントを獲得しているか
- **ハッシュタグ別の効果**：どのハッシュタグを使った投稿のリーチが広いか

**アプローチ2：API経由でデータを自動取得**

Xは公式APIでツイートのパフォーマンスデータを取得できます（Basicプランは月額$200、Proプランは月額$5,000。2026年2月以降、従量課金（Pay-Per-Use）モデルも導入されています。※最新価格は公式サイトで確認してください）。Instagramも、Instagram Graph APIを通じてビジネスアカウントのインサイトデータを取得可能です。ただし、APIの設定にはやや技術的な知識が必要です。本書の読者層を考慮すると、まずはアプローチ1から始めることをおすすめします。

### 改善提案レポートの自動生成

データを集計するだけでなく、「次にどうすればよいか」を提案するレポートを自動生成させましょう。これこそが、SNS運用代行の付加価値です。

エージェントのシステムプロンプトに、以下のようなレポートフォーマットを指定します。

```
## 月次SNS運用レポート

### 1. 全体サマリー
- 総投稿数：XX件
- 平均エンゲージメント率：X.X%
- 前月比：+X.X%（↑/↓）
- フォロワー増減：+XX人

### 2. トップ投稿ベスト3
1. [投稿内容の要約]（エンゲージメント率：X.X%）
   → 好調の要因分析：...
2. ...
3. ...

### 3. 改善が必要な投稿ワースト3
1. [投稿内容の要約]（エンゲージメント率：X.X%）
   → 改善提案：...

### 4. 来月への提案
- コンテンツ面：...
- 投稿時間帯：...
- ハッシュタグ戦略：...
```

このレポートをクライアントに毎月提出するだけで、「データに基づいた運用改善を提案してくれるプロ」として評価されます。レポート作成にかかる時間は、エージェントなら5〜10分です。

エンゲージメント分析エージェントのシステムプロンプト例を示します。

```
あなたはSNSエンゲージメント分析の専門家です。

## 役割
クライアントのSNSアカウントの投稿パフォーマンスデータを分析し、改善提案を含む月次レポートを作成する。

## 入力
- CSVファイル（投稿日時、投稿内容、インプレッション、いいね、コメント、リポスト）

## 作業手順
1. CSVデータをBashツール（Python/pandas）で読み込む
2. エンゲージメント率を計算（(いいね+コメント+リポスト)÷インプレッション×100）
3. カテゴリー別・時間帯別・ハッシュタグ別の集計を実施
4. トップ投稿・ワースト投稿を特定し、要因を分析
5. 上記のレポートフォーマットに従い、月次レポートを作成
6. 結果をMarkdownファイルに書き出す

## 制約
- 数値は小数点第1位まで表示
- 前月比較が可能な場合は必ず記載
- 改善提案は具体的なアクション（「投稿数を増やす」ではなく「火曜・木曜の19時台に1投稿ずつ追加する」）にする
```

## 7-4. コスト試算：3アカウント運用代行を自動化する場合

ここまで紹介した3つのエージェント（投稿作成・カレンダー生成・エンゲージメント分析）を使って、3つの企業アカウントの運用代行を行う場合のコストと収益を計算する。

### 運用代行の相場 vs エージェント運用コスト

**収入側：運用代行の報酬**

フリーランスのSNS運用代行の相場は、1アカウントあたり月額3万〜10万円です。ここでは1アカウント月額5万円×3件 = 月額15万円の収入を想定します。

**支出側：エージェント運用コスト**

各エージェントのセッション時間とコストを見積もります。

| エージェント | 1回のセッション時間 | 月間実行回数 | 月間セッション時間 |
|------------|-------------------|------------|-----------------|
| 投稿作成 | 約3分/投稿 | 36回（3アカウント×月12投稿） | 108分 |
| カレンダー生成 | 約5分/アカウント | 3回 | 15分 |
| エンゲージメント分析 | 約8分/レポート | 3回 | 24分 |
| **合計** | | | **147分** |

- セッション時間課金：147分（約2.5時間） × $0.08/時間 = $0.20（約30円）
- API利用料（目安）：月額約$15〜20（約2,300〜3,000円）
- **合計コスト：月額約2,330〜3,030円**

**ROI（投資対効果）**

- 月収：150,000円
- 月間コスト：約3,000円
- 粗利：約147,000円
- ROI：5,000%

### 作業時間削減と品質維持のバランス

エージェント導入前後の作業時間を比較する。

**手動の場合（3アカウント合計）**
- 投稿文作成：月12投稿 × 3アカウント × 30分 = 18時間
- コンテンツカレンダー作成：3アカウント × 2時間 = 6時間
- エンゲージメント分析・レポート：3アカウント × 3時間 = 9時間
- **合計：月33時間**

**エージェント活用の場合**
- エージェントへの指示・設定確認：3アカウント × 30分 = 1.5時間
- 生成された投稿文の確認・修正：月36投稿 × 5分 = 3時間
- カレンダーの確認・微調整：3アカウント × 15分 = 0.75時間
- レポートの確認・カスタマイズ：3アカウント × 20分 = 1時間
- **合計：月6.25時間**

作業時間は月33時間から約6時間へ、**約81%の削減**です。浮いた27時間で、さらに2〜3アカウントの案件を受注することも可能です。

ただし、「すべてをエージェント任せにする」のは禁物です。以下のポイントは人間が必ず確認しましょう。

- **投稿文のブランド整合性**：エージェントが生成した文章が、クライアントのブランドイメージに合っているか
- **センシティブなトピック**：時事ネタや社会問題に触れる投稿は、炎上リスクがないか人間が判断
- **クライアントへの報告**：レポートの数字は正確か、提案は的を射ているか

エージェントは「優秀なアシスタント」であって、「責任者」ではありません。最終的な品質保証は、あなたの仕事です。



### ハンズオン10：X向け投稿文バッチ生成エージェントをつくる

**ゴール**: 1つのテーマから、X向けの投稿文を5本まとめて（バッチで）生成するエージェントを作成する

**準備**: Anthropic Consoleにログイン済み、antコマンドが使える状態

1. Consoleの「Agents」→「Create Agent」を開き、以下を設定します。
   - **Name**: `x-post-generator`
   - **Model**: `claude-sonnet-4-6`
   - **Tools**: `Web search`, `Write`を有効化
   - **System Prompt**: 以下をコピペしてください

   ```
   あなたはX（旧Twitter）の投稿文を作成するSNSライターです。

   ## 基本ルール
   - 日本語で140字以内の投稿文を生成する
   - 1回のリクエストで5本の投稿文を生成する
   - 各投稿は「主張→根拠→CTA」の3行構成とする
   - 絵文字は1投稿あたり1〜2個まで
   - ハッシュタグは1投稿あたり2個まで（文字数に含む）

   ## 作業手順
   1. ユーザーからテーマとターゲット層を受け取る
   2. Web searchでそのテーマの最新トレンドを調査する
   3. トレンドを反映した5パターンの投稿文を生成する
   4. 各投稿の文字数を確認し、140字以内に収める
   5. 結果をMarkdown形式でファイルに書き出す

   ## 出力フォーマット
   各投稿は以下の形式で出力：

   **投稿1**（XX字）
   [投稿本文]

   **狙い**: [この投稿で意図するエンゲージメントの種類]
   **推奨投稿時間**: [曜日と時間帯]

   ```

2. 「Environments」で、インターネットアクセスをONにしたEnvironmentを選択（または新規作成）します。

3. ターミナルからセッションを作成します。以下のコマンドの`agent_id`と`environment_id`を自分のものに置き換えてください。

   ```bash
   ant sessions create \
     --agent-id agent_xxxxxxxx \
     --environment-id env_xxxxxxxx \
     --message "テーマ：副業でのAI活用。ターゲット：30代会社員。5本の投稿文を生成して、results/x_posts.md に保存してください"
   ```

4. セッション完了後、結果を確認します。

   ```bash
   ant sessions get --session-id session_xxxxxxxx
   ```

5. 出力ファイルを確認します。5本の投稿文がそれぞれ140字以内で生成されていればOKです。

**確認ポイント**: 
- 5本すべてが140字以内に収まっているか
- 各投稿にハッシュタグが含まれているか

**SNS自動投稿の注意点 — 各プラットフォームの利用規約を確認する**


エージェントでSNS投稿文を「作成」するのは何の問題もありません。しかし、「自動で投稿する」段階では、各プラットフォームの利用規約に注意が必要です。

**X（旧Twitter）のルール**

Xは、公式APIを使った自動投稿を認めています。ツイートの予約投稿やRSS連携も許可されています。ただし、以下は明確に禁止されています。

- 自動フォロー・自動アンフォロー
- トレンドトピックへの自動投稿によるトレンド操作
- 自動DM送信（マーケティング目的）
- API以外の方法（スクレイピング等）での自動操作

AI生成コンテンツの投稿自体は禁止されていません。ChatGPTやClaudeで作成したツイートをAPIで投稿することは、規約上問題ありません。

**Instagramのルール**

Instagramは、Instagram Graph APIを通じたビジネスアカウントへの投稿を公式に許可しています。ただし、以下の点に注意が必要です。

- 自動いいね・自動フォローは明確に禁止（アカウント停止のリスク）
- 投稿頻度に暗黙の制限があり、短時間に大量投稿するとアカウントが制限される場合がある
- API経由でもリールの投稿には一部制限がある

**noteのルール**

noteは2026年4月時点で、公式の投稿APIを公開していません。非公式のAPIを解析して自動投稿するツールが一部のエンジニアによって開発されていますが、利用規約に抵触するリスクがあります。noteへの投稿は、エージェントに「下書きのMarkdownファイル」を生成させ、人間がnoteのエディタにコピペして投稿するのが安全な運用方法です。

**本書での推奨アプローチ**

本書で構築するエージェントは、投稿文の「作成」に集中します。実際の投稿操作は、以下のいずれかの方法で行ってください。

1. **手動投稿**：エージェントが生成した投稿文をコピペして各プラットフォームに投稿
2. **公式ツール活用**：XのTweetDeckやMetaのBusiness Suiteで予約投稿
3. **連携サービス**：Buffer、Hootsuite等のSNS管理ツールに投稿文を登録

「全自動で投稿まで」を目指すのではなく、「投稿文の作成と計画を全自動化し、投稿ボタンだけ人間が押す」という運用が、規約違反のリスクを避けつつ最大の効率化を実現する方法です。

## まとめ

- **SNS投稿作成エージェントで、プラットフォームに最適化された投稿文を自動生成できる**。X・Instagram・noteの文体・文字数・構成パターンをシステムプロンプトで定義し、テーマを渡すだけで投稿文が完成する

- **自動「投稿」ではなく自動「作成」にフォーカスすることが、規約違反を避ける鍵**。投稿文の生成と計画はエージェントに任せ、実際の投稿は人間が行う運用が最も安全で効率的
